P-V2 Code-Smell checking

Code smells are symptoms in the software source code that possibly indicate deeper problems. Similarly tools such as OOPS27 can be used for vocabulary smell checking. OOPS is a Web-based tool for detecting common ontology pitfalls such as: (1) missing relationships; (2) using incorrectly ontology elements and (3) missing domain and range properties. The complete list of pitfalls that are detected by OOPS is presented in [Poveda-Villalón et al., 2012].
